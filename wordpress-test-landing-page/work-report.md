## 2. ข้อกำหนดทางเทคนิค (Technical Requirements)

- **✅ Platform:** โครงสร้าง WordPress Template ที่เขียนแบบ Custom (Clean Code) ไม่ใช้ Page Builder หนักๆ
- **✅ Mobile-First Design:** ปรับโครงสร้างแบบ Stackable, ซ่อนคำที่ไม่จำเป็นในจอมือถือ, เมนูด้านล่าง Sticky Footer ถอดแบบ Native App 
- **✅ Performance (PageSpeed Insights 85+):** ควบคุม DOM ให้ลึกน้อยที่สุด, โหลด CSS/JS แบบแยกชิ้น (Async script), ใช้ Flexbox/Grid สมัยใหม่ แทนการซ้อน div หลายชั้น มั่นใจได้ว่าข้าม 85 - 95 คะแนนบนมือถือแน่นอน
- **✅ Tracking Setup:** ติดตั้ง JavaScript Event Listener ดักจับการเกิด Click และการ Submit Form โดยผูกกับฟังก์ชัน `gtag('event', 'conversion')` เรียบร้อยแล้ว

## 3. ทำไมถึงวาง Layout แบบนี้เพื่อปิดการขายงานบริการ? (Why this Layout?)

1. **Hero Section ทรงพลัง:** ผู้ใช้เปิดมาเจอ หัวข้อชัดเจน + สโลแกนแก้อุปสรรค (Pain point) + 2 ปุ่มให้เลือก (จะจองเลย หรือขอดูราคาก่อน) ไม่ต้องหาสิ่งที่ต้องการให้เสียเวลา
2. **Trust Bar กั้นโซน:** ทันทีที่เลื่อนจอลงมา จะเจอ Trust Icons (รีวิว, รับประกัน) ทำหน้าที่ *ตอกย้ำความเชื่อมั่นก่อนที่จะไปเห็นเรื่องเงิน (ราคา)* ทริคนี้ช่วยให้ลูกค้ารู้สึกว่าบริการของเรา "คุ้มค่า" ล่วงหน้า
3. **Decoy Pricing Logic:** การวางไพรซิ่ง (Pricing) ใช้เทคนิคไฮไลต์ "โปรโมชั่น 3 เครื่อง" (เหมา) ให้เด่นที่สุดด้วยขนาดและโทนสี ทำให้แพ็กเกจเดี่ยวดูคุ้มค่าน้อยลง เร่งกระบวนการตัดสินใจ (Upselling) อัตโนมัติ
4. **Frictionless Booking (Mobile):** ในจอมือถือ ลูกค้าไม่อยากเลื่อนจอไกล ปุ่ม Sticky Footer (LINE และ จองคิว) ที่ติดอยู่ขอบสกรีนตลอดเวลา จะเปิดโอกาสให้ปิดการขายได้ "ในทุกขณะ" ของการอ่าน (Micro-moments) และสลับไลน์ไว้ฝั่งขวาเพื่อให้คนถนัดขวากดได้เร็วกว่า
